# 🎮 파이프 시프터 (Pipe Shifter) - 스테이지 개발 및 설계 가이드

이 문서는 다른 AI 모델이 **파이프 시프터(Pipe Shifter)** 게임의 동작 원리, 물리/수학적 규칙, 코드 구조를 즉시 이해하고 **기존 스테이지(1~3)를 검증하거나 새로운 스테이지(4, 5, 6 등)를 추가**할 수 있도록 돕는 지침서입니다.

---

## 1. 게임 개요
- **목표**: 초등학교 4학년 '평면도형의 이동(밀기, 뒤집기, 돌리기)' 교육과정을 시각적 퍼즐로 이해하도록 함.
- **플랫폼**: 웹 표준 정적 페이지 (HTML5, Vanilla JS, CSS3)
- **승리 조건**: 시작 파이프(수도꼭지)부터 도착 파이프(물통)까지 물이 새지 않고 연결되도록 플레이어 파이프들을 배치하고 조작함. 밸브를 열었을 때 물이 끝까지 도달하면 스테이지 클리어.

---

## 2. 핵심 게임 메커니즘 & 조작법
플레이어는 파이프를 클릭하여 다음과 같은 **평면도형 이동 조작**을 할 수 있습니다.

1. **밀기 (Slide / 평행이동)**
   - 파이프 클릭 시 상/하/좌/우의 비어 있는 격자가 초록색 하이라이트로 활성화됩니다.
   - 초록색 칸을 누르면 파이프가 그 자리로 미끄러지듯 이동합니다. (장애물이나 다른 파이프를 뛰어넘을 수 없으며 직선으로만 이동 가능)
2. **돌리기 (Rotate / 회전이동)**
   - 조작 패널의 좌우 회전 버튼으로 파이프를 시계 방향 또는 반시계 방향으로 **90도씩 회전**시킵니다.
3. **뒤집기 (Flip / 대칭이동)**
   - 좌우 대칭 버튼으로 파이프를 거울에 비친 것처럼 뒤집습니다. (수학적으로 `scaleX * -1` 대칭 구현)

---

## 3. 기술적 규칙 및 수학적 검증 로직 (AI 필독)

스테이지를 생성할 때 AI가 반드시 지켜야 할 **연결구(Openings) 연산 방식**입니다.

### A. 방향 정의 (Directions)
모든 연결 방향은 다음과 같이 `0`부터 `3`까지의 숫자로 정의됩니다.
- `0`: 위 (Top)
- `1`: 오른쪽 (Right)
- `2`: 아래 (Bottom)
- `3`: 왼쪽 (Left)

### B. 파이프 종류 및 기본 연결구 (pipeBaseOpenings)
회전 및 뒤집기를 하지 않은 기본 상태(0도 회전, 뒤집기 없음)일 때 각 파이프가 가진 연결 방향 배열입니다.
- **`I`형 파이프**: `[0, 2]` (위, 아래로 열림)
- **`L`형 파이프**: `[0, 1]` (위, 오른쪽으로 열림)
- **`T`형 파이프**: `[1, 2, 3]` (오른쪽, 아래, 왼쪽으로 열림)
- **`CROSS`형 파이프**: `[0, 1, 2, 3]` (사방으로 열림)
- **`ASYM` (비대칭 L자형)**: `[0, 1]` (비대칭 시각 디자인 적용)

### C. 실시간 연결구 변화 계산 공식
게임 엔진(`game.js`)이 파이프의 조작 상태에 따라 최종 연결구 방향을 계산하는 공식입니다.

1. **대칭이동 (Flip)** 적용:
   - 좌우 뒤집기(`flipX === -1`)인 경우: 왼쪽(`3`)과 오른쪽(`1`)을 서로 바꿈.
   - 상하 뒤집기(`flipY === -1`)인 경우: 위(`0`)와 아래(`2`)를 서로 바꿈.
2. **회전이동 (Rotation)** 적용:
   - 위에서 계산된 방향값 `o`에 회전 상태 값 `rot` (0, 1, 2, 3 중 하나)를 더한 뒤, 4로 나눈 나머지 값을 구함.
   - **최종 방향** = `(o + rot) % 4`

### D. 두 파이프 간의 연결 판정 조건
타일 A(x, y)와 인접한 타일 B가 특정 방향 `d`로 맞닿아 있을 때:
- A의 최종 연결구 배열에 `d`가 포함되어 있어야 함.
- B의 최종 연결구 배열에 반대 방향인 `(d + 2) % 4`가 포함되어 있어야 함.
- 이 조건이 만족하면 두 파이프는 물이 흐를 수 있게 연결된 것입니다.

## 4. 스테이지 데이터 구조 (`levels.js` 정의용)

AI는 `levels.js` 파일 내의 `stages` 배열에 아래 객체 구조로 스테이지를 설계 및 추가해야 합니다.

### 데이터 스키마
```javascript
{
    cols: 5,              // 격자 열 개수 (자유롭게 확장 가능, 예: 6)
    rows: 5,              // 격자 행 개수 (자유롭게 확장 가능, 예: 6)
    optimalMoves: 7,      // 3성 클리어를 위한 모범 조작 횟수 (이동, 회전, 대칭 횟수의 총합)
    pipes: [
        // 1. 시작점 (수도꼭지 - 무조건 고정 및 start 속성 필요)
        { id: 'start', type: 'L', x: 0, y: 0, rot: 1, isFixed: true, start: true },
        
        // 2. 도착점 (물통 - 무조건 고정 및 target 속성 필요)
        { id: 'end', type: 'I', x: 4, y: 4, rot: 0, isFixed: true, target: true },
        
        // 3. 움직일 수 있는 플레이어 파이프 (isFixed: false)
        // [중요] 디버그 및 검증을 위한 answer(정답 상태)를 반드시 명시해야 합니다.
        { 
            id: 'p1', 
            type: 'I', 
            x: 1, y: 0,    // 게임 시작 시 초기 배치 좌표 (겹치지 않아야 함)
            rot: 0,        // 초기 회전 상태 (0~3)
            isFixed: false,
            answer: { 
                x: 1, y: 0, // 정답 위치
                rot: 1      // 정답 회전 상태
            } 
        },
        ...
    ]
}
```

## 5. 다음 AI 모델을 위한 미션 (Mission for the next AI)

이 문서를 읽은 다음 AI 모델은 아래 미션을 수행해야 합니다.

1. **기존 스테이지 (1, 2, 3) 검증**:
   - `levels.js`에 정의된 각 스테이지의 `pipes`들의 `answer` 조합을 적용했을 때, 시작(faucet)에서 끝(target)까지의 물길이 막힘없이 성공적으로 연결되는지 DFS/BFS 경로상으로 수학적 검증을 완료하십시오.
2. **신규 스테이지 추가 (4, 5, 6 등)**:
   - 단계를 확장하여 새로운 스테이지 데이터를 설계하십시오.
   - **Stage 4 (고정 파이프 테마)**: 움직이지 못하는 `isFixed: true` 상태의 부식된 고정 파이프가 맵 중간에 배치되며, 플레이어가 고정 파이프의 입출구 방향에 맞춰서 자신의 파이프를 조작해야 하는 퍼즐.
   - **Stage 5 (복합 복잡 미로)**: `T`자형 파이프 혹은 격자 크기를 늘리고(예: 6x6), '뒤집고 돌리기'와 '돌리고 뒤집기'의 순서 차이를 명확히 체득해야만 풀리는 고난도 종합 스테이지.
   - 모든 새로운 스테이지에는 **반드시 유효한 `answer` 객체와 합리적인 `optimalMoves`를 설계**해야 합니다.
3. **코드 적용**:
   - 새로 작성한 스테이지 데이터를 `levels.js`의 `stages` 배열 뒤에 이어 붙여주십시오.
